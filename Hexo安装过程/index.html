# 搭建 Hexo

# 前提提要

首先需要一下软件

  • Nodejs
  • hexo 插件
  • Git

# 安装

# 1. 安装 nodejs

官网链接：https://nodejs.org/en/

注意：左侧是稳定版、右边是测试版。一般都选稳定版

# 1.2 测试
按 [win] + r 输入cmd 
输入node -v 查看node的版本
输入npm -v  查看npm的版本

# 1.3 更改为淘宝镜像源

npm install -g cnpm --registry=https://registry.npm.taobao.org

# 2. 安装 hexo

cnpm install -g hexo-cli
# 2.1 测验
输入hexo -v 查看heox 版本

# 3 安装 Git

下载链接：https://gitee.com/all-about-git
一路无脑安装即可
# 3.1 测试
右键打开 Git Bash Here
能够成功打开即可说明安装成功

# 创建

# 1. 创建文件夹

直接在你想要创建博客的地方新建一个文件夹 [blog]

# 2. 进入文件夹，初始化 hexo

在该文件夹中打开cmd
输入hexo init进行初始化（可能会有点卡，懂？）

# 3. 本地启动 hexo 博客

一般在发布到服务器之前，会先在本地进行测试，测试成功后再发布到服务器上
在文件夹中打开cmd
输入hexo s / hexo server（二选一即可，只不过是一个缩写，一个全写）

启动完成后，请不要关闭cmd窗口，否则浏览器会访问不到
然后打开浏览器，输入localhost:4000

# 4. 新建文章

可以直接在cmd中输入hexo n "创建的文章名称"

注意：也可以直接在 blog 文件夹下打开 \source\_posts 下新建一个 markdown 文件即可
若需要访问图片，需要另外在 typroa 中进行偏好设置

# 推送到远端服务器

# 1. 修改配置文件

在 blog 文件夹下打开 _config.yml ，直接拉到最下面编辑一下内容

# 2. 安装插件

在blog文件夹下打开cmd
npm install --save hexo-deployer-git

# 3. 推送到远端

在blog文件夹下打开cmd
hexo clean 清除缓存
hexo g	   生成文件
hexo d	   推送到远端

# 4. 设置远端使用网页服务

Gitee：[Setting]——>[Gitee Page]——> 开启

# 更换主题

在 blog 文件夹下打开 _config.yml

# 详细教程

教程链接：https://shoka.lostyu.me/computer-science/note/theme-shoka-doc/
